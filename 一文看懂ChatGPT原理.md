# 一文看懂 ChatGPT 原理：从一个"起名字"的小实验说起

> 你不需要会写代码，也能完全看懂这篇文章。
>
> AI 大神 Andrej Karpathy（前 OpenAI 联合创始人、前特斯拉 AI 总监）用 200 行代码写了一个迷你版 ChatGPT，能学会给人起名字。这篇文章会用最通俗的方式，带你彻底理解 ChatGPT 到底是怎么工作的。

---

## 先说结论：ChatGPT 在干什么？

一句话：**ChatGPT 就是一个超大规模的"猜下一个字"游戏**。

你打出"今天天气真"，它猜下一个字可能是"好"（概率 60%）、"热"（概率 20%）、"冷"（概率 15%）……然后从中选一个，再继续猜下一个字，一个字一个字地"蹦"出来，直到它觉得说完了。

就这么简单。

它之所以看起来"很聪明"，是因为它读了整个互联网的文本，见过几乎所有人类写过的句子模式，所以它"猜"得极其准确。

为了让你真正理解这个过程，Karpathy 做了一个极简版的实验——不猜句子，只猜**人名**。让我们从这个小实验开始。

---

## 第一章：给模型一本"教材"

### 教材是什么？

一份包含 32000 个英文人名的列表，一行一个：

```
emma
olivia
ava
isabella
sophia
...
```

模型的任务：**读完这些名字后，学会自己编新名字**。

这些新名字不是从列表里抄的，而是模型理解了"英文名字的规律"之后，自己创造出来的。比如它可能编出"kamon"、"karai"、"elina"这样的词——听起来像真名字，但查不到任何真人叫这个名。

### 这跟 ChatGPT 有什么关系？

关系非常直接：

| | 迷你实验 | ChatGPT |
|---|---|---|
| 教材 | 3.2 万个人名 | 整个互联网的文本 |
| 学什么 | 名字的字母规律 | 人类语言的所有规律 |
| 能干什么 | 编新名字 | 写文章、回答问题、写代码…… |

原理完全一样，只是规模不同。就像一辆玩具车和一辆真正的汽车——发动机原理是一样的，只是大小和马力不同。

## 第二章：翻译——把文字变成数字

### 为什么要变成数字？

计算机的本质是计算器，它只认识数字，不认识字母。所以第一步，我们需要一个"翻译官"，把每个字母翻译成一个数字编号。

这个翻译官有个专业名字叫**分词器（Tokenizer）**。

### 怎么翻译？

很简单，给每个出现过的字母发一张"身份证"：

| 字母 | 编号 |
|:---:|:---:|
| a | 0 |
| b | 1 |
| c | 2 |
| ... | ... |
| z | 25 |
| **[开始/结束]** | **26** |

最后一个编号 26 是个特殊标记，叫 **BOS**（Beginning of Sequence，序列开始）。它的作用就像一本书的封面和封底——告诉模型"名字从这里开始"和"名字到这里结束"。

### 举个例子

"emma" 这个名字被翻译成：

```
[开始]  →  e  →  m  →  m  →  a  →  [结束]
  26      4     12    12     0      26
```

一共 27 种可能的符号（26 个字母 + 1 个特殊标记），这就是模型的全部"词汇量"。

> **对比 ChatGPT**：ChatGPT 的词汇量大约是 10 万个"词块"（不是单个字母，而是常见的字母组合，比如"ing"、"tion"会被当作一个词块）。但翻译的原理完全一样——把文字变成数字。

### 一个重要的细节

这些编号之间**没有大小关系**。4 号不比 2 号"更大"，就像你的身份证号码比别人大，不代表你比别人"更高级"。它们只是不同的标签，用来区分不同的字母。

## 第三章：核心游戏——猜下一个字母

### 游戏规则

这是整个 ChatGPT 的灵魂，也是最容易理解的部分：

**给模型看前面的字母，让它猜下一个是什么。**

还是以 "emma" 为例，模型要做 5 次猜测：

| 第几次 | 模型能看到的 | 正确答案 |
|:---:|:---:|:---:|
| 第 1 次 | [开始] | e |
| 第 2 次 | [开始] e | m |
| 第 3 次 | [开始] e m | m |
| 第 4 次 | [开始] e m m | a |
| 第 5 次 | [开始] e m m a | [结束] |

就像**完形填空**——给你一段话，让你填下一个字。

### 这跟 ChatGPT 聊天有什么关系？

完全一样的游戏！当你问 ChatGPT "中国的首都是"，它看到这几个字，然后：
- 猜第一个字："北"（概率 99%）
- 看到"中国的首都是北"，猜下一个："京"（概率 99%）
- 看到"中国的首都是北京"，猜下一个："。"（概率很高）

一个字一个字往外蹦，直到它觉得"说完了"。

**所有的语言模型——从这个迷你实验到 ChatGPT——都在玩同一个游戏：猜下一个。**

---

## 第四章：模型怎么"猜"——给每个选项打分

### 第一步：打原始分

模型看到前面的字母后，会给所有 27 种可能的下一个字母各打一个分。

这些分数可以是任何数字——正的、负的、大的、小的。比如：

```
a: 1.2 分    b: -0.5 分    c: 0.3 分    ...    e: 3.8 分    ...    z: -1.2 分
```

这些原始分数叫 **Logits**——你可以理解为考试的"卷面原始分"，还没有换算成百分比。

### 第二步：把分数变成概率

原始分可正可负、大小不一，没法直接使用。我们需要把它们变成**概率**——每个选项占百分之几，所有选项加起来正好 100%。

这个转换过程叫 **Softmax**（软最大化），你可以把它想象成一场**选举计票**：

1. 先把所有人的票数变成正数（通过一个数学运算，分数越高的会被**放大**得越多）
2. 然后算每个人占总票数的百分比

举个例子：

| 字母 | 原始分 | → Softmax 后 |
|:---:|:---:|:---:|
| e | 3.8 分 | **60%** |
| a | 1.2 分 | 22% |
| m | 0.8 分 | 10% |
| 其他24个字母 | 各种低分 | 合计 8% |

注意：分数高的选项会**碾压**分数低的。3.8 分和 1.2 分看起来差距不大，但转成概率后变成了 60% 对 22%。这是因为 Softmax 会**放大差异**——让模型能"果断地做出选择"。

### 第三步：按概率抽签

得到概率后，模型**不是直接选概率最高的**，而是按概率**随机抽签**。

60% 的概率选 "e"，22% 的概率选 "a"，10% 的概率选 "m"……

这就像一个转盘抽奖——"e" 占了转盘的 60%，大多数时候会中，但偶尔也会转到别的格子。这样生成的名字才有**多样性**，不会每次都一模一样。

## 第五章：模型的"大脑"长什么样

好了，现在你知道游戏规则了。但关键问题来了：**模型凭什么能打出合理的分数？** 它的"大脑"里到底有什么？

### 参数：4192 个旋钮

模型的全部"知识"就存储在 **4192 个数字**里，这些数字叫**参数（Parameters）**。

你可以把模型想象成一个巨大的调音台，上面有 4192 个旋钮。每个旋钮的位置（数字的大小）决定了模型的行为。

- 一开始，所有旋钮都是**随机拨的**——模型什么都不懂，就像刚出生的婴儿
- 训练的过程就是**慢慢调整每个旋钮**，让模型越来越准
- 训练结束后，这些旋钮的位置就是模型学到的全部"知识"

> **对比 ChatGPT**：ChatGPT 有多少个旋钮？大约**数千亿个**。从 4192 到数千亿——这就是"迷你版"和"完整版"的区别。但原理完全一样。

### 这些旋钮具体是干什么用的？

这 4192 个旋钮被组织成几组，各有分工：

**第一组：字母身份证（嵌入层）**

前面说过，字母 "e" 的编号是 4。但一个简单的数字 "4" 信息量太少了，模型没法用它做复杂的判断。

所以模型给每个字母准备了一份 **16 个数字组成的"性格档案"**。比如：

```
字母 "a" 的档案：[0.12, -0.03, 0.45, ..., 0.08]  （16个数字）
字母 "e" 的档案：[0.31, 0.15, -0.22, ..., 0.44]  （16个数字）
```

这些数字一开始是随机的，但训练之后，模型会自己学到有意义的模式。比如：
- 元音字母（a, e, i, o, u）的档案可能变得很相似
- 经常出现在名字开头的字母，档案可能有某些共同特征

**没有人告诉模型"a 和 e 都是元音"，它自己从数据中发现了这个规律。** 就像小孩学说话——没人教他语法规则，但他听多了自然就会了。

这个"把编号变成档案"的过程，叫 **嵌入（Embedding）**。

**第二组：座位号（位置嵌入）**

同一个字母 "a" 出现在名字开头和名字结尾，含义是不同的。所以除了"字母档案"，还有一份"位置档案"：

```
位置 0（第一个字）的档案：[0.05, 0.33, ..., -0.11]  （16个数字）
位置 1（第二个字）的档案：[-0.02, 0.18, ..., 0.27]  （16个数字）
```

最终，一个字母的"完整信息"= **字母档案 + 位置档案**，合并成一个 16 个数字的向量。

就像一个学生走进教室，他既有"我是谁"的信息，也有"我坐在哪"的信息。

**第三组：注意力的连接线路（后面详细说）**

**第四组：独立思考的线路（后面详细说）**

## 第六章：注意力机制——字母之间怎么"交流"

这是整个 Transformer 架构（GPT 的底层架构）最核心的创新，也是最值得理解的部分。

### 为什么需要"交流"？

想象你在猜 "emma" 的最后一个字母。如果你只看到"当前位置的前一个字母是 m"，你很难猜。但如果你能**回头看看前面所有的字母** "e-m-m"，你就更有把握猜出下一个是 "a"。

**注意力机制就是让模型在预测时，能够"回头看"前面所有的字母，并决定重点关注哪些。**

### 图书馆找书的类比

想象一个图书馆的场景：

你（当前位置的字母）带着一个**搜索关键词**走进图书馆。图书馆里的每本书（前面的每个字母）封面上都有一个**标签**，书里面有**实际内容**。

1. 你拿着**搜索关键词**（Query），跟每本书的**封面标签**（Key）一一对比
2. 对比的结果是一个**相关性分数**——跟你的关键词越匹配，分数越高
3. 把这些分数变成**注意力权重**（用 Softmax）——最相关的书得到最大的权重
4. 最后，按权重把所有书的**实际内容**（Value）混合起来——最相关的书贡献最多

用专业术语说，每个字母会生成三样东西：

| 名称 | 含义 | 类比 |
|:---:|:---:|:---:|
| **Query**（查询） | "我在找什么信息？" | 你手里的搜索关键词 |
| **Key**（钥匙） | "我能提供什么线索？" | 书封面上的标签 |
| **Value**（值） | "如果被选中，我提供什么内容？" | 书的实际内容 |

### 一个重要限制：只能看过去，不能看未来

每个位置的字母**只能关注它前面的字母**，不能偷看后面的。

这很好理解——当模型在生成名字时，是一个字母一个字母往后写的。它在写第 3 个字母时，第 4 个字母还不存在。

这就像你在写一封信，每写下一个字，都只能参考你**已经写过的内容**，不能参考还没写的部分。

### 多头注意力：同时用多个视角看问题

模型不只有一套"搜索系统"，而是同时有 **4 套**（在迷你实验中是 4 个，ChatGPT 中有几十上百个）。

每一套叫一个"**注意力头**"（Attention Head），就像 4 个研究员同时去图书馆，每个人带着不同的研究课题：

- **研究员 A**：专门关注最近几个字母（"刚才出现了什么？"）
- **研究员 B**：专门关注名字的开头（"这个名字是什么风格的？"）
- **研究员 C**：专门寻找元音的模式（"元音和辅音的节奏对不对？"）
- **研究员 D**：关注重复出现的字母（"有没有双字母？"）

4 个研究员各自找到有用的信息后，**把发现汇总**，形成对当前位置的完整理解。

> **这些"研究课题"不是人为指定的。** 模型在训练过程中自己学会了让不同的注意力头关注不同的模式。就像一个团队合作久了，每个人自然会找到自己擅长的分工。

## 第七章：独立思考——MLP 前馈网络

注意力做完之后，每个位置的字母已经收集了来自其他位置的信息。但信息还需要**消化加工**。

这就是 **MLP（多层感知机）** 的工作，你可以理解为模型的"独立思考时间"。

如果说注意力是"开会讨论、收集各方意见"，那 MLP 就是"会后自己坐下来琢磨"。

具体过程：

1. **展开**：把信息从 16 维展开到 64 维（放大 4 倍），就像把一个压缩包解压，展开所有细节
2. **过滤**：用一个叫 **ReLU** 的操作过滤——如果某个维度的值是负数，就直接变成 0（扔掉），正数保持不变。这就像一个"信息过滤器"，只保留有价值的正向信号
3. **压缩**：再从 64 维压回 16 维，得到精炼后的信息

## 第八章：两个精妙的设计

### 残差连接：信息的"快车道"

想象你在玩传话游戏——第一个人说一句话，经过 10 个人传递后，往往面目全非。

在模型里，信息经过层层处理后也会"失真"。**残差连接**的做法是：

**处理后的结果 = 原始信息 + 新处理得到的信息**

就像传话游戏中，每个人不仅把自己理解的版本传下去，还把**原话也一并传下去**。这样即使中间有人理解偏了，原始信息也不会丢失。

没有残差连接，深度神经网络根本训练不起来——信息会在传递中彻底消散。这个看似简单的"加法"，是深度学习发展史上最重要的发明之一。

### 归一化：自动音量调节器

在信息传递过程中，数字可能变得越来越大或越来越小。如果不控制，计算就会"爆炸"或"消失"。

**RMSNorm（归一化）** 就像一个自动音量调节器：
- 如果音量太大（数字太大），就调小
- 如果音量太小（数字太小），就调大
- 让信号始终保持在一个合理的范围内

就像调音师在录音棚里做的事——不管歌手唱得多大声或多小声，最终输出的音量总是稳定的。

## 第九章：完整的信息流水线

现在把所有零件组装起来！一个字母在模型里经过的完整流程如下：

```
 ① 字母 "m"（在名字"emma"的第3个位置）
          ↓
 ② 嵌入：查"m的性格档案" + 查"位置3的座位号" → 合并成一个16维向量
          ↓
 ③ 归一化：调节音量，保持稳定
          ↓
 ④ 注意力：回头看前面的 [开始]、e、m，收集它们的信息
   （4个注意力头同时工作，各自关注不同模式）
          ↓
 ⑤ 残差连接：注意力的结果 + 第③步的原始信息
          ↓
 ⑥ 归一化：再次调节音量
          ↓
 ⑦ MLP：独立思考，展开→过滤→压缩
          ↓
 ⑧ 残差连接：MLP的结果 + 第⑤步的信息
          ↓
 ⑨ 输出层：把16维向量变成27个分数（每个字母一个分数）
          ↓
 ⑩ Softmax：把分数变成概率
          ↓
 结果：a=45%, [结束]=30%, o=8%, e=5%, ...
```

整个过程不到一秒就完成了。

> **在迷你实验中**，这条流水线只有 1 层（步骤④~⑧只走一遍）。
> **在 ChatGPT 中**，有几十到上百层——步骤④~⑧要重复几十到上百遍，每一层都在上一层的基础上进一步加工信息。层数越多，模型理解就越深。

## 第十章：衡量错误——模型猜得有多离谱？

模型猜了一个概率分布，我们知道正确答案是什么。现在需要一个**打分方式**来告诉模型它猜得好不好。

这个打分方式叫**交叉熵损失（Cross-Entropy Loss）**，听起来吓人，其实很直观：

**损失 = 模型觉得正确答案有多"意外"**

- 模型给正确答案 90% 的概率 → 不意外 → 损失很小（0.1）
- 模型给正确答案 1% 的概率 → 非常意外 → 损失很大（4.6）
- 模型给正确答案 100% 的概率 → 完全不意外 → 损失为 0

打个比方：老师出了一道题，学生不仅要**答对**，还要**有信心地答对**。如果学生只是蒙对了（给正确答案只分配了很小的概率），老师照样给低分。

**训练的目标就是让损失越来越小——让模型越来越有信心地给出正确答案。**

一开始，模型完全是瞎蒙（27 个选项随机猜），损失约为 3.3。训练结束后，损失降到 2.37 左右——模型已经学到了很多规律。

## 第十一章：反向传播——追查每个旋钮的责任

现在模型猜完了，我们知道它猜得有多差（损失）。下一个问题是：**怎么改进？**

模型有 4192 个旋钮。要改进，需要知道：**每个旋钮应该往哪个方向拧，拧多少？**

### 多米诺骨牌的类比

想象一长串多米诺骨牌：

- **正向过程**（Forward）：推倒第一张牌，一路连锁反应，最终倒下最后一张牌（得到损失值）
- **反向过程**（Backward）：从最后一张牌**倒推回去**，追查每张牌对最终结果的"影响力"有多大

这个倒推的过程就叫**反向传播（Backpropagation）**——深度学习最核心的算法。

### 追查事故责任的类比

公司年底亏了 100 万（损失）。老板想知道：**哪个部门、哪个员工应该对亏损负最大的责任？**

反向传播做的就是这件事：

1. 先画出公司的组织架构图（整个计算过程的路径）
2. 从亏损出发，一层层往下追
3. 在每一层，用一个叫**链式法则**的方法分配责任：
   - **员工的责任 = 员工对部门的影响力 × 部门对公司的影响力**

追查完毕后，每个旋钮（参数）都会得到一个**责任分数**，这个分数叫**梯度（Gradient）**。

梯度告诉我们两件事：
- **方向**：这个旋钮应该往上拧还是往下拧
- **力度**：应该拧多少

就像一个指南针，指向"让错误变小"的方向。

---

## 第十二章：调整旋钮——Adam 优化器

知道了每个旋钮的责任（梯度）之后，最简单的做法是：哪个方向能让损失变小，就往那个方向拧一点。

但这种简单的做法有问题——有时候会来回震荡，走弯路。

所以工程师们发明了一个更聪明的调整策略，叫 **Adam 优化器**。你可以把它想象成一个**有记忆的登山者**在浓雾中下山：

- **普通登山者**：每走一步都只看脚下的坡度，往最陡的方向走。问题是如果地形复杂，可能来回打转。
- **Adam 登山者**：
  - 记住最近几步的**平均方向**（惯性）——如果一直往东走，即使某一步地形说该往西，也不会急转弯，避免来回震荡
  - 记住最近几步的**颠簸程度**（经验）——如果某段路很颠簸，就走慢点、走稳点；如果很平坦，就放心大步走

此外，还有一个**学习率**的概念——相当于每一步的"步幅"：

- 学习率太大：步子迈太大，可能跨过最优点，来回跳
- 学习率太小：步子太小，走得太慢，训练要很久
- 这个实验的策略：**开始步子大，越到后面步子越小**（线性衰减）。就像开车接近目的地时减速。

---

## 第十三章：训练全过程——一个名字一个名字地学

万事俱备！现在把所有环节串起来。训练过程就是一个**大循环**，重复 1000 次，每次做这些事：

### 每一轮的完整故事

> **第 1 步：取一个名字**
>
> 从 32000 个名字中取一个，比如 "emma"。
>
> **第 2 步：翻译成数字**
>
> "emma" → [26, 4, 12, 12, 0, 26]（开始 + e + m + m + a + 结束）
>
> **第 3 步：逐个位置预测**
>
> 在每个位置，模型看到前面的字母，预测下一个。5 个位置就做 5 次预测。
>
> **第 4 步：计算损失**
>
> 把 5 次预测的错误程度取平均值，得到这个名字的总损失。
>
> **第 5 步：反向传播**
>
> 从损失出发倒推，算出每个旋钮的责任（梯度）。
>
> **第 6 步：调整旋钮**
>
> 用 Adam 优化器，根据梯度微调每个旋钮，让下次预测更准。
>
> **第 7 步：清零，准备下一轮**
>
> 把所有梯度清零，取下一个名字，回到第 1 步。

### 训练过程中能看到什么？

随着训练的进行，模型生成的名字质量会肉眼可见地提高：

| 训练阶段 | 生成的名字示例 | 解读 |
|:---:|:---:|:---:|
| 刚开始（第 1 轮） | xqbzjfmwplk | 完全乱码，瞎猜 |
| 早期（第 100 轮） | brrtos, kloey | 有点结构了，但还很奇怪 |
| 中期（第 500 轮） | kamon, karai | 有模有样了 |
| 后期（第 1000 轮） | anna, anton, elina | 完全像真名字！ |

你能亲眼看到模型在**"开窍"**：

- 一开始它不知道任何规律，随便拼字母
- 慢慢地它发现"辅音后面通常跟元音"
- 再后来它学会了"名字通常 3-8 个字母长"
- 最后它掌握了完整的英文名字统计规律

**这就是"学习"——不需要有人告诉它规则，它自己从大量数据中总结出规律。**

---

## 第十四章：生成新名字——让模型自由发挥

训练完成！现在轮到模型"创作"了。

### 生成过程

1. 给模型一个"开始"信号
2. 模型输出 27 个概率（下一个字母可能是什么）
3. 按概率随机选一个字母
4. 把这个字母喂回去，模型再预测下一个
5. 重复，直到模型输出"结束"信号
6. 把中间选出的字母拼起来，就是一个新名字！

### 温度：创意程度的调节旋钮

生成时有一个重要的设置叫**温度（Temperature）**，它控制模型的"创意程度"：

| 温度值 | 效果 | 类比 |
|:---:|:---:|:---:|
| **0.1**（很低） | 几乎只选概率最高的字母 | 一个拘谨的人，只敢说最安全的话 |
| **0.5**（适中） | 大部分时候选高概率的，偶尔冒险 | 一个成熟的人，稳重但有灵感 |
| **1.0**（标准） | 完全按学到的概率分布选 | 一个自然的人，该怎样就怎样 |
| **2.0**（很高） | 低概率的选项也有很大机会被选中 | 一个疯狂的艺术家，什么都敢试 |

**温度太低**：生成的名字很"安全"但无聊，总是那几个最常见的（"anna"、"john"）。

**温度太高**：生成的名字很"大胆"但可能是乱码（"xkqwm"）。

**对于这个实验，0.5 是最佳点**——既有创意，又不会太离谱。

> **你在 ChatGPT 设置里看到的"创意模式"和"精确模式"，本质上就是在调温度。** 精确模式 = 低温度，创意模式 = 高温度。

## 第十五章：从迷你实验到 ChatGPT——差的只是规模

现在你已经完全理解了这个迷你实验的原理。那么问题来了：**从这个小实验到真正的 ChatGPT，差距在哪？**

答案令人惊讶——**核心算法完全一样**，差的只是**规模和效率**。

| 维度 | 迷你实验 | ChatGPT |
|:---:|:---:|:---:|
| 训练数据 | 3.2 万个名字 | 整个互联网的文本（几万亿字） |
| 词汇量 | 27 个字符 | 约 10 万个子词 |
| 参数量 | 4,192 个 | 数千亿个 |
| 层数 | 1 层 | 几十到上百层 |
| 注意力头 | 4 个 | 几十到上百个 |
| 向量维度 | 16 维 | 数千维到上万维 |
| 硬件 | 你的笔记本电脑 | 数千块 GPU 组成的集群 |
| 训练时间 | 几分钟 | 几个月 |
| 训练成本 | 零 | 几千万到数亿美元 |

但**循环是完全一样的**：

```
分词 → 嵌入 → 注意力 → 独立思考 → 预测下一个字
     → 衡量错误 → 反向传播 → 调整参数 → 重复
```

ChatGPT 之所以能写论文、回答问题、写代码，不是因为它用了什么神秘的算法，而是因为：

1. **它见过足够多的文本**——几乎所有维基百科、书籍、论坛、代码库……
2. **它有足够多的参数来记忆这些模式**——数千亿个旋钮可以编码极其复杂的语言规律
3. **它有足够多的层来做深度思考**——几十上百层处理，信息被反复加工提炼

**量变引起质变。** 当"猜下一个字"的游戏被放大到如此规模时，模型涌现出了令人惊叹的能力——这就是所谓的"智能涌现"。

## 术语对照表

如果你在别的文章里看到这些术语，现在你知道它们是什么了：

| 术语 | 一句话解释 | 对应的比喻 |
|:---:|:---|:---|
| **Token** | 文本被切成的最小单位（字母或词块） | 字母积木 |
| **Tokenizer** | 把文字翻译成数字的工具 | 翻译官 |
| **Embedding** | 把编号变成一组有意义的数字 | 性格档案 |
| **Parameter** | 模型存储知识的数字 | 调音台上的旋钮 |
| **Attention** | 让模型回头看前面的内容 | 图书馆找书 |
| **Multi-Head Attention** | 多套注意力同时工作 | 多个研究员各自找资料 |
| **Query / Key / Value** | 注意力机制的三个组件 | 搜索关键词 / 书的标签 / 书的内容 |
| **MLP** | 每个位置独立加工信息 | 开完会后自己思考 |
| **Residual Connection** | 保留原始信息不丢失 | 传话时同时传原话 |
| **RMSNorm** | 让数字大小保持稳定 | 自动音量调节器 |
| **Softmax** | 把分数变成概率 | 选举计票 |
| **Logits** | 模型输出的原始分数 | 考试原始卷面分 |
| **Loss** | 模型猜得有多差 | 考试扣了多少分 |
| **Cross-Entropy** | 一种衡量预测错误的方式 | 正确答案有多"意外" |
| **Gradient** | 每个参数应该怎么调 | 指南针 |
| **Backpropagation** | 倒推每个参数的责任 | 追查事故责任 |
| **Adam** | 聪明的参数调整策略 | 有记忆的登山者 |
| **Learning Rate** | 每次调整的步幅 | 登山的步伐大小 |
| **Temperature** | 控制生成的创意程度 | 创意旋钮 |
| **Inference** | 训练完后让模型生成内容 | 让学生做题/创作 |
| **Transformer** | GPT 的底层架构 | 整条信息加工流水线 |

## 写在最后

现在你知道了——**ChatGPT 并不神秘**。

它的核心就是一个不断重复的简单循环：看前文 → 猜下一个字 → 对照答案 → 调整自己 → 再来。

200 行代码就能写出这个算法的完整版本。剩下的，真的"只是效率"——用更快的硬件、更大的数据、更多的参数，让这个简单的游戏达到令人难以置信的效果。

> **"Everything else is just efficiency."**
>
> ——Andrej Karpathy
